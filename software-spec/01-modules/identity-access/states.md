# Состояния

## Пользовательский доступ

```text
ГОСТЬ
  ├─ почта/пароль ───────────────► ПОЛНАЯ СЕССИЯ
  ├─ Telegram с внешней связью ──► ПОЛНАЯ СЕССИЯ
  └─ Telegram без достаточной связи ► ОГРАНИЧЕННАЯ СЕССИЯ

ОГРАНИЧЕННАЯ СЕССИЯ
  ├─ ключ доступа создан ─────────► ПОЛНАЯ СЕССИЯ
  └─ выход/истечение ─────────────► ГОСТЬ
```

## Почта

`не привязана → ожидает подтверждения → подтверждена`; изменение подтверждённого адреса возвращает состояние к ожиданию подтверждения. Уже подтверждённый адрес повторно подтверждается без изменения результата.

## Сессия

`активна → короткая часть истекла → обновлена`; либо `активна → отозвана/истекла`. Обнаружение повторного использования переводит связанную сессию в отозванную.

## Объединение

`ожидает решения → выполняется → завершено`; альтернативы: отменено, истекло или завершилось ошибкой. Временная аренда выполнения не позволяет двум исполнителям одновременно менять владельца.
