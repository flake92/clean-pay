# Критерии приёмки

- Неверная рабочая конфигурация не позволяет приложению начать обслуживать запросы.
- Простая проверка жизни не зависит от БД или внешних систем.
- Публичная готовность не раскрывает внутренние сообщения и положительна только для свежего успешного снимка.
- Подробный интерфейс нельзя отличить от отсутствующего без правильного секрета.
- Зависшие фоновые исполнители обнаруживаются по сердцебиению.
- Очистка соблюдает отдельные сроки и не удаляет активные сессии или платежные данные.
- Журналы не содержат секретных значений.
- Устанавливаемое приложение не кэширует персональные данные и корректно показывает автономный экран.
- Текущая desktop/mobile оболочка воспроизведена в макете.
