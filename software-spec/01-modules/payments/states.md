# Состояния

```text
ГОТОВА
  │ исполнитель получил исключительное право
  ▼
ОТПРАВЛЯЕТСЯ
  ├─ доказанный успех ───────────► УСПЕШНА
  ├─ доказанная конечная ошибка ► ЗАВЕРШЕНА_ОШИБКОЙ
  └─ результат не доказан ──────► ИСХОД_НЕИЗВЕСТЕН

ИСХОД_НЕИЗВЕСТЕН
  ├─ найден платёж ─────────────► УСПЕШНА
  ├─ доказана ошибка ───────────► ЗАВЕРШЕНА_ОШИБКОЙ
  ├─ безопасный повтор разрешён ► ГОТОВА
  └─ предел автоматизации ──────► РУЧНАЯ_ПРОВЕРКА
```

Внешняя запись платежа имеет состояния: ожидает, завершён, ошибка, отменён, возвращён, неизвестен. Они не заменяют состояние команды: команда отвечает за безопасность отправки, запись — за известный результат транзакции.
