# Сценарии

## Новая платёжная команда

1. Проверить тело и сессию.
2. Найти существующую операцию по пользователю и ключу.
3. Если её нет, применить ограничение частоты, доказать внешнего владельца и создать операцию.
4. Получить исключительный захват.
5. Перечитать предложение и сравнить все подтверждённые условия.
6. Зафиксировать отправку.
7. Создать платёж во внешней системе с серверным адресом возврата.
8. Проверить ответ, сохранить платёж и вернуть его пользователю.

## Повтор

Сохранённый успех или ошибка возвращаются без внешней отправки. Выполняемая операция возвращает 202. Ручная проверка возвращает 409 с запретом повторной оплаты.

## Сверка

Исполнитель захватывает ограниченное число подходящих операций, проверяет точный результат по пользовательскому или административному интерфейсу, обновляет запись и назначает следующую попытку. Незавершённая история обрабатывается отдельным ограниченным шагом.
