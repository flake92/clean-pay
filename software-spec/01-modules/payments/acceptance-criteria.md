# Критерии приёмки

- Два одновременных запроса с одним ключом не создают два внешних платежа.
- Повторный успешный ответ имеет тот же идентификатор операции и признак повтора.
- Другой отпечаток с тем же ключом отклоняется.
- Изменившаяся цена/валюта/версия не оплачивается без нового подтверждения.
- Клиентский адрес возврата не влияет на внешний запрос.
- Неопределённый исход переживает перезапуск и не превращается в автоматическую повторную покупку.
- Окончательный локальный результат читается при внешней недоступности.
- Объединение пользователей не переносит платёж под другого внешнего владельца.
- Ручная проверка явно запрещает повтор и показывает идентификатор для поддержки.
- История сохраняет порядок, пустой результат и совместимый старый режим.
