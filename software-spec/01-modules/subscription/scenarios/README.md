# Сценарии подписки и доступа

## SUB-SC-001. Публичные тарифы и персональные предложения

HTTP-021 отдаёт публичные тарифы без сессии. HTTP-023 требует полного пользователя и возвращает персональные предложения. Пустой массив является штатным результатом; ошибка внешней системы преобразуется по общему контракту и не подменяется фиктивным тарифом.

## SUB-SC-002. Текущая подписка

1. HTTP-022 получает подписку Remnashop.
2. Отсутствие подписки возвращается как штатное пустое состояние.
3. Для существующей подписки система ищет в Remnawave ровно одного пользователя, совпавшего по устойчивому идентификатору либо однозначному сочетанию e-mail/Telegram.
4. Только `ACTIVE`, не истёкшая и однозначно принадлежащая пользователю запись даёт live URL. Сохранённый upstream URL не служит запасным.
5. Неизвестный внешний статус передаётся как неизвестный; ошибка сопоставления не превращается в активный доступ.

## SUB-SC-003. Устройства

HTTP-028 перечисляет устройства. HTTP-029 удаляет все устройства отдельной командой; HTTP-030 удаляет ровно один URL-кодированный `hwid`. Повтор имеет семантику Remnashop, локальной гарантии идемпотентности нет. PAGE-008 независимо отображает загрузку, пустой список, таблицу/карточки, mutation pending и ошибку.

## SUB-SC-004. Перевыпуск и промокод

HTTP-026 перевыпускает ссылку; HTTP-027 применяет промокод. Обе операции требуют полной сессии, rate/source/CSRF-защиту общего слоя и аудит. Результат не вычисляется оптимистично: интерфейс обновляется только после ответа и повторного чтения актуального состояния.

## SUB-SC-005. Деградация

Недоступность Remnashop не возвращает устаревший персональный объект как новый факт. Недоступность Remnawave позволяет показать прочие данные подписки, но операция, требующая подтверждённого URL, получает безопасное отсутствие/конфликт. Ошибка одной панели кабинета не скрывает успешно загруженные независимые панели.

## Проверки

Проверяются отсутствие подписки, active/expired/unknown, неоднозначные совпадения Remnawave, пустые устройства, неверный и URL-значимый `hwid`, повтор удаления, неизвестный промокод, rate limit, потеря внешней сессии и независимая деградация панелей.
