# Бизнес-правила

1. Публичные тарифы доступны без пользовательской сессии.
2. Персональные предложения, подписка и управление устройствами требуют полного разрешённого пользователя и действующей внешней связи.
3. Отсутствие текущей подписки является штатным пустым результатом.
4. Для существующей подписки показывается только однозначная действующая ссылка из системы фактического доступа.
5. Внешняя сохранённая ссылка подписки не используется как запасной вариант, если фактическую ссылку подтвердить нельзя.
6. Пользователь считается тем же владельцем ссылки только при совпадении основного идентификатора либо однозначном согласовании доступных почты и Telegram.
7. Неизвестные состояния подписки сохраняются и показываются, а не преобразуются в активное состояние.
8. Удаление одного устройства адресует ровно один аппаратный идентификатор; удаление всех устройств — отдельная операция.
9. Перевыпуск ссылки, удаление устройств и промокод являются изменяющими действиями и обязательно журналируются.
10. Повтор изменяющей операции без отдельного ключа имеет семантику внешней системы; Clean Pay не обещает идемпотентность, которой нет в контракте.
