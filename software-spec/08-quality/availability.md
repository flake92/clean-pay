# Доступность и деградация

| Отказ | Требуемое поведение |
|---|---|
| PostgreSQL недоступен | readiness `degraded`; защищённые/мутирующие операции не выдают ложный успех |
| Redis недоступен | readiness `degraded`; локальный/долговечный механизм лимита продолжает защиту согласно конкретной операции; кэш готовности не считается свежим |
| Remnashop недоступен | операции, требующие live-данных, возвращают нормализованную upstream-ошибку; локальный терминальный платёжный результат можно вернуть без нового dispatch |
| Remnawave недоступен/нет совпадения | безопасное отсутствие результата внутри интеграции и бизнес-конфликт 409 для требующей URL операции |
| Telegram JWKS/OIDC недоступен | вход не завершается; state не считается успешно использованным без доказанной подписи |
| SMTP недоступен | операция следует точному контракту частичного успеха; подтверждённое владение не откатывается из-за последующей доставки/merge |
| Turnstile недоступен | публичное защищаемое действие не обходится молча, если проверка обязательна |
| Worker остановился | HTTP-приложение может жить, но readiness/операционный мониторинг показывает просроченный heartbeat и растущую очередь |

Приложение, БД, Redis и очистка имеют автоматический перезапуск; данные переживают перезапуск в volumes. Не заявляется числовой SLA. Совместимый минимум — ограниченные тайм-ауты, явная деградация, долговечность неизвестных платёжных исходов и отсутствие двойного внешнего эффекта.
