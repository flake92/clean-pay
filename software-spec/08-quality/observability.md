# Качество наблюдаемости

Каждый входящий запрос получает события «получен» и «решение» на proxy, кроме явно исключённых статических assets. Каждый внешний запрос фиксирует method, очищенный path, status и duration; аутентификация — этапы церемонии; мутации/аудит — смысловое действие. Тесты обязаны доказывать редактирование и отсутствие query/token/password/cookie. Оператор связывает платёж по локальному operation ID и безопасным correlation headers, а не по секретному внешнему ключу. Каталог и приёмники заданы в `07-operations/observability.md`.
