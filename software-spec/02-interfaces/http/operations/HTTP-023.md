# Получить персональные предложения

## Идентификатор

`HTTP-023`

## Назначение

Вернуть доступные текущему аккаунту тарифы, длительности и способы оплаты.

## Владелец

Модуль подписки.

## Акторы

Вошедший пользователь.

## Предусловия

Полная разрешённая сессия и Remnashop-связь.

## Логический входной контракт

Входных данных нет.

## Текущий транспорт

`GET /subscription/offers`; server-rendered collection page для вошедшего пользователя.

## Правила валидации

Внешний ответ обязан соответствовать RS-014: `gateways[]`, `plans[]`, `has_current_subscription`, `current_subscription_status`.

## Нормализация

Порядок массивов, регистр строк и значения передаются без локальной сортировки или исправления.

## Авторизация

Локальная сессия и access token Remnashop.

## Идемпотентность

Чтение повторяемо, но предложение не резервирует цену.

## Основной сценарий

Выполнить RS-014 и передать точный объект server-rendered view.

## Альтернативные сценарии

Пустые `gateways` и `plans` допустимы.

## Ошибочные сценарии

`401`, `403`, нормализованные `4xx`; внешний `5xx`/сеть/таймаут — `502`; неверный ответ — `502`; локальная ошибка — `500`.

## Логический результат

`200 text/html`; Rails рендерит доступные продление, тарифы и способы оплаты.

## Побочные эффекты

Только возможный refresh сессии и журналы.

## Транзакционные требования

Нет бизнес-транзакции.

## Наблюдаемость

Журналируется внешний статус и latency; содержимое предложения не содержит секретов, но целиком не логируется.

## Источники

Доказательства находятся в `09-traceability/`; внешний контракт RS-014.

## Статус уверенности

`требует повторной проверки после ADR-003`
