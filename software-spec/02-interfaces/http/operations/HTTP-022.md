# Получить текущую подписку и живую ссылку

## Идентификатор

`HTTP-022`

## Назначение

Вернуть подписку текущего владельца, заменив сохранённый адрес подключения актуальным адресом Remnawave.

## Владелец

Модуль подписки.

## Акторы

Вошедший пользователь с разрешённым доступом.

## Предусловия

Полная сессия, подтверждённый e-mail по политике доступа, действующая Remnashop-связь.

## Логический входной контракт

Входных данных нет; тело и query игнорируются.

## Текущий транспорт

`GET /subscription`; server-rendered singleton resource, полная Rails-сессия обязательна.

## Правила валидации

Ответ RS-013 — `null` либо полный объект подписки. Для непустого объекта поиск Remnawave выполняется по `user_remna_id` с контрольными e-mail и Telegram ID; адрес принимается только по правилам RW-001.

## Нормализация

Все поля Remnashop сохраняются, поле `url` безусловно заменяется живым Remnawave URL. Сохранённый Remnashop URL клиенту не раскрывается.

## Авторизация

Локальная сессия и access token Remnashop; при необходимости токены обновляются и cookie ротируются.

## Идемпотентность

Логическое чтение; возможна ротация сессии/внешних токенов.

## Основной сценарий

Получить RS-013, найти соответствующего пользователя в Remnawave, вернуть подписку с живым URL.

## Альтернативные сценарии

RS-013 возвращает `null` — Rails рендерит пустое состояние подписки, обращение к Remnawave не выполняется.

## Ошибочные сценарии

`401 UNAUTHORIZED`, `403 FORBIDDEN`, `404 SUBSCRIPTION_NOT_FOUND`; отсутствие однозначного живого URL — `409 SUBSCRIPTION_URL_UNAVAILABLE`; Remnashop `5xx`/сеть/таймаут — `502`; ошибки Remnawave — по RW-001; локальная ошибка — `500`.

## Логический результат

`200 text/html`; Rails рендерит текущую подписку либо её пустое состояние.

## Побочные эффекты

Возможны refresh токенов, ротация cookie и технические журналы; подписка не изменяется.

## Транзакционные требования

Обновление локальных токенов и session cookie выполняется по контракту сессий; два внешних чтения не образуют транзакцию.

## Наблюдаемость

Различаются сбой Remnashop, отсутствие subscription и сбой/неоднозначность Remnawave; URL и токены не журналируются.

## Источники

Доказательства находятся в `09-traceability/`; внешние контракты RS-013 и RW-001.

## Статус уверенности

`требует повторной проверки после ADR-003`
