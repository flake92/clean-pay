# Запросить код подтверждения e-mail

## Идентификатор

`HTTP-007`

## Назначение

Попросить Remnashop создать и отправить одноразовый шестизначный код на выбранный адрес.

## Владелец

Модуль доступа и идентичности.

## Акторы

Вошедший пользователь с подтверждённым или ещё неподтверждённым e-mail.

## Предусловия

Восстанавливаемая локальная сессия и доступная внешняя учётная запись; при включённом Turnstile — действующий token.

## Логический входной контракт

| Поле | Тип | Обязательно | Nullable | Формат | Ограничения | По умолчанию | Назначение |
|---|---|---:|---:|---|---|---|---|
| `email` | string | нет | нет | e-mail | внешняя схема trim/lower-case, ≤255 | выбранный Remnashop pending/current e-mail | адрес назначения |
| `turnstileToken` | string | нет | нет | непрозрачный | приоритетный token | нет | антибот |
| `cf-turnstile-response` | string | нет | нет | непрозрачный | fallback | нет | антибот |

Поля Turnstile удаляются; остальные дополнительные поля уходят Remnashop и его закреплённой схемой игнорируются.

## Текущий транспорт

`POST /api/bff/auth/email/request-verification`; доверенный origin; `application/json`/`application/*+json`; JSON-объект UTF-8 до 65 536 байт; cookie сессии. Query/path-параметров нет.

## Правила валидации

Общее тело проверяет Clean Pay; `email` проверяет Remnashop. Turnstile выполняется первым. Правый крайний валидный IP из `X-Forwarded-For` передаётся Turnstile как `remoteip`; остальные IP headers игнорируются.

## Нормализация

Rate-limit нормализует идентичность trim/lower-case и HMAC-хэширует её. Внешнюю нормализацию адреса выполняет Remnashop.

## Авторизация

Proxy специально разрешает `/api/bff/auth/email/*` неподтверждённой e-mail-сессии. Сервер получает внешние токены с `allowUnverifiedEmail=true`.

## Идемпотентность

Нет: успешный вызов создаёт новый код и письмо. Cooldown запрещает второй запуск в течение 60 секунд для пользователя.

## Основной сценарий

1. Проверить запрос и Turnstile.
2. Восстановить локальную/внешнюю сессию.
3. Увеличить cooldown-счётчик `email_verification_request` с лимитом 1/60 секунд по user ID.
4. Увеличить общий счётчик той же action с лимитом 5/900 секунд по входному e-mail либо локальному e-mail и Telegram ID.
5. Вызвать RS-009.
6. Записать audit с фактическим `target_email`.

## Альтернативные сценарии

Только ошибка `UPSTREAM_UNAVAILABLE`, чьё исходное сообщение содержит `failed to send verification email`, повторяется до трёх общих попыток: ожидание 300 мс перед второй и 600 мс перед третьей. Любая иная ошибка — без повтора. Из-за сетевой неопределённости письмо могло быть принято SMTP до ошибочного ответа.

## Ошибочные сценарии

| HTTP | Код | Условие |
|---:|---|---|
| 400 | `VALIDATION_ERROR` | тело/e-mail недопустимы |
| 401 | `UNAUTHORIZED`/`EMAIL_REQUIRED` | нет сессии или внешней связи |
| 403 | `FORBIDDEN` | origin/Turnstile |
| 413 | `VALIDATION_ERROR` | слишком большое тело |
| 415 | `VALIDATION_ERROR` | media type |
| 429 | `RATE_LIMITED` | cooldown, общий или внешний предел |
| 502 | `UPSTREAM_UNAVAILABLE`/`UPSTREAM_ERROR` | Remnashop/SMTP после разрешённых повторов |
| 503 | `UPSTREAM_UNAVAILABLE` | Turnstile недоступен |
| 500 | `INTERNAL_ERROR` | локальная сессия, счётчик или аудит |

`retryAfterSeconds` присутствует только в debug metadata непроизводственного ответа; заголовок `Retry-After` не устанавливается.

## Логический результат

`200`:

```json
{"data":{"success":true,"target_email":"user@example.com","expires_at":"2026-07-22T12:15:00Z"}}
```

Все три поля обязательны по внешнему контракту; cookie специально не меняются, кроме возможной общей refresh rotation.

## Побочные эффекты

Счётчики Redis, письмо SMTP через Remnashop, состояние хэша кода/срока во внешней системе, audit и технические журналы.

## Транзакционные требования

Счётчики, отправка письма, внешнее сохранение кода и локальный audit не образуют одну транзакцию. Remnashop сохраняет код только после успешной SMTP-передачи.

## Наблюдаемость

Фиксируются этапы и адрес назначения; сам код и token не журналируются.

## Источники

Доказательства находятся в `09-traceability/`.

## Статус уверенности

`подтверждено`
