# Выполнить совместимый выход

## Идентификатор

`HTTP-040`

## Назначение

Сохранить старый transport-контракт выхода.

## Владелец

Платформенный модуль совместимости.

## Акторы

Любой браузер, включая уже вышедший.

## Предусловия

Trusted source для browser mutation.

## Логический входной контракт

Нет; тело не читается.

## Текущий транспорт

Публичный bodyless `POST /api/logout`; Origin/Referer policy; cookies при наличии.

## Правила валидации

Content-Type не требуется. Origin/Referer проверяется edge guard.

## Нормализация

Ответ без `data`, `{ok:true}`.

## Авторизация

Сессия не обязательна.

## Идемпотентность

Повтор безопасен.

## Основной сценарий

Записать audit `auth_logout`, отозвать/очистить web session, удалить cookie, вернуть ok.

## Альтернативные сценарии

Нет действующей сессии — cookie всё равно очищаются и возвращается успех, если storage operation успешна.

## Ошибочные сценарии

`403` source guard. Ошибка audit/session до ответа — необработанный framework 500; в отличие от HTTP-005 handler не использует BFF error mapper.

## Логический результат

`200 {"ok":true}`.

## Побочные эффекты

Audit, отзыв сессии и Set-Cookie delete.

## Транзакционные требования

Audit выполняется до clear session; общей транзакции нет.

## Наблюдаемость

Action без токена/cookie value.

## Источники

Доказательства находятся в `09-traceability/`; новый контракт — HTTP-005.

## Статус уверенности

`подтверждено`
