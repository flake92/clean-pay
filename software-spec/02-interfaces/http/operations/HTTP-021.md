# Получить публичные тарифы

## Идентификатор

`HTTP-021`

## Назначение

Показать каталог тарифов до входа пользователя.

## Владелец

Модуль подписки.

## Акторы

Анонимный или вошедший пользователь.

## Предусловия

Нет.

## Логический входной контракт

Входных данных нет. Тело и параметры строки запроса не используются.

## Текущий транспорт

`GET /api/bff/plans/public`; cookie и пользовательская авторизация не требуются.

## Правила валидации

Локальная валидация отсутствует. Внешний ответ обязан соответствовать контракту RS-012.

## Нормализация

Ответ внешней системы передаётся без преобразования внутри оболочки `data`.

## Авторизация

Отсутствует.

## Идемпотентность

Чтение повторяемо; содержимое каталога может измениться между запросами.

## Основной сценарий

Выполнить RS-012 без access token и вернуть результат.

## Альтернативные сценарии

Пустой каталог является успешным результатом.

## Ошибочные сценарии

`400/404/409/429` внешней системы нормализуются по общей таблице Remnashop; внешний `5xx`, сетевой сбой и таймаут дают `502 UPSTREAM_UNAVAILABLE`; неверный контракт — `502 UPSTREAM_ERROR`; непредвиденная локальная ошибка — `500`.

## Логический результат

`200 {"data":<публичный каталог Remnashop>}`.

## Побочные эффекты

Только технические журналы исходящего и входящего HTTP.

## Транзакционные требования

Нет.

## Наблюдаемость

Журналируются путь, метод, latency и итоговый статус без пользовательских секретов.

## Источники

Доказательства находятся в `09-traceability/`; внешний wire-контракт — RS-012.

## Статус уверенности

`подтверждено`
