# Перевыпустить ссылку подписки

## Идентификатор

`HTTP-026`

## Назначение

Инвалидировать прежнюю и получить новую внешнюю ссылку подключения.

## Владелец

Модуль подписки.

## Акторы

Вошедший пользователь.

## Предусловия

Полная разрешённая сессия и Remnashop-связь.

## Логический входной контракт

Входных данных нет; присланное тело обработчиком не читается.

## Текущий транспорт

`POST /subscription/reissue`; bodyless Rails form mutation с CSRF и полной сессией.

## Правила валидации

Только общие session/origin policy и проверка внешнего ответа `{success:boolean}`.

## Нормализация

Нет.

## Авторизация

Локальная сессия и access token Remnashop.

## Идемпотентность

Ключ отсутствует; повтор может перевыпустить ссылку повторно.

## Основной сценарий

Записать attempted audit, выполнить RS-017 без тела, записать succeeded audit, вернуть результат.

## Альтернативные сценарии

`success:false` является успешным HTTP-ответом, если так ответила внешняя система.

## Ошибочные сценарии

`401/403/404/409/429`, `502` для upstream/контракта, `500`. Если failed-audit сам не записался, наружу может уйти ошибка аудита вместо исходной. Если succeeded-audit падает после успешного RS-017, клиент получает ошибку при уже перевыпущенной ссылке; автоматической компенсации нет.

## Логический результат

`303 See Other` на `/subscription` с flash-результатом.

## Побочные эффекты

Изменение внешней подписки и три возможных audit action: `subscription_reissue_attempted|succeeded|failed`.

## Транзакционные требования

Внешнее изменение и audit не атомарны.

## Наблюдаемость

Audit содержит user ID, но не URL.

## Источники

Доказательства находятся в `09-traceability/`; внешний контракт RS-017.

## Статус уверенности

`требует повторной проверки после ADR-003`
