# Удалить все устройства подписки

## Идентификатор

`HTTP-029`

## Назначение

Освободить все занятые слоты устройств текущей подписки.

## Владелец

Модуль подписки.

## Акторы

Вошедший пользователь.

## Предусловия

Полная разрешённая сессия и Remnashop-связь.

## Логический входной контракт

Нет; тело не читается.

## Текущий транспорт

`DELETE /api/bff/subscription/devices`; trusted origin; session cookie.

## Правила валидации

Общий origin/session gate; ответ RS-020 `{success:boolean}`.

## Нормализация

Нет.

## Авторизация

Локальная сессия и access token Remnashop.

## Идемпотентность

Ключ отсутствует; повтор должен считаться допустимым пользовательским действием, но конкретный результат задаёт Remnashop.

## Основной сценарий

Attempted audit, RS-020 без тела, succeeded audit, возврат результата.

## Альтернативные сценарии

`success:false` передаётся как успешный ответ.

## Ошибочные сценарии

`401/403/404`; любой внешний ответ ошибки на devices отображается в `DEVICE_DELETE_UNAVAILABLE`, upstream `5xx` — статус `409`; сетевой сбой — `502`; локальная/audit ошибка — `500`. Успешное удаление может предшествовать ошибке succeeded-audit.

## Логический результат

`200 {"data":{"success":boolean}}`.

## Побочные эффекты

Удаление внешних устройств и audit `devices_delete_all_*`.

## Транзакционные требования

Внешняя мутация и audit не атомарны.

## Наблюдаемость

User ID/action/outcome без HWID.

## Источники

Доказательства находятся в `09-traceability/`; RS-020.

## Статус уверенности

`подтверждено`
