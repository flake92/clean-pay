# Получить контакты поддержки

## Идентификатор

`HTTP-033`

## Назначение

Дать интерфейсу настроенные каналы поддержки.

## Владелец

Платформенный модуль.

## Акторы

Вошедший пользователь.

## Предусловия

Пограничная политика требует полную разрешённую сессию; сам handler пользователя не читает.

## Логический входной контракт

Нет.

## Текущий транспорт

`GET /support`; публичный server-rendered Rails resource.

## Правила валидации

Конфигурация приложения должна быть валидна; optional значения уже trim-нормализованы при загрузке.

## Нормализация

Ключи ответа camelCase: `enabled,email,telegramUsername,faqUrl`.

## Авторизация

Только edge/session gate; handler не делает повторной проверки.

## Идемпотентность

Чтение.

## Основной сценарий

Rails читает типизированную конфигурацию и рендерит опубликованные contact-card либо явное пустое состояние.

## Альтернативные сценарии

Каналы могут быть `null`, а `enabled` — false.

## Ошибочные сценарии

`303` на страницу входа при отсутствии сессии; `500` при ошибке конфигурации/handler.

## Логический результат

`200 text/html`; Rails рендерит доступные каналы поддержки и их пустые состояния.

## Побочные эффекты

Только access log.

## Транзакционные требования

Нет.

## Наблюдаемость

Не логировать e-mail/Telegram целиком сверх конфигурационного аудита.

## Источники

Доказательства находятся в `09-traceability/`; внешний смысл каналов — `04-integrations/support-channels.md`.

## Статус уверенности

`требует повторной проверки после ADR-003`
