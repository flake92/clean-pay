# Служебные и совместимые HTTP-контракты

Доказательства: TRACE-HTTP-033—037, 040, 044.

## HTTP-033 — Поддержка

- `GET /support`; параметров и тела нет; по текущей пограничной политике требуется сессия.
- 200 `text/html`; Rails рендерит только опубликованные e-mail, Telegram и FAQ либо явное пустое состояние.
- Ошибки: `303` на вход при отсутствии сессии либо 500 конфигурации. Данные не изменяются.

## HTTP-034 — Базовая проверка жизни

- `GET /health`; публично.
- Всегда при работающем процессе: 200 `{"status":"ok","service":"clean-pay","version":string}`.
- Зависимости не проверяются.

## HTTP-035 — Проверка жизни

- `GET /health/liveness`; публично.
- Тот же точный ответ 200, что HTTP-034; назначение — проверка процесса оркестратором.

## HTTP-036 — Публичная готовность

- `GET /health/readiness`; публично, без кэширования HTTP.
- Тело: `{status:"ok"|"degraded",checkedAt:string|null,stale:boolean,service:"clean-pay",version:string}`.
- 200 только для свежего успешного снимка; 503 для отсутствующего, устаревшего или отрицательного снимка.

## HTTP-037 — Подробная готовность

- `GET /internal/health/readiness` с `x-clean-pay-readiness-secret`.
- Неверный секрет: 404 `{"error":{"code":"NOT_FOUND","message":"Not found"}}`.
- Верный секрет: `{status,checkedAt,checks,service,version}`; каждая проверка содержит `status:"ok"|"down"`, `latencyMs:number`, необязательное `message`.
- 200 при полном успехе, 503 при деградации; `cache-control:no-store`.
- Неожиданная ошибка: 503 `{"status":"degraded","service":"clean-pay","checkedAt":null}` без подробностей.

## HTTP-044 — Служебный сценарий веб-приложения

- `GET /service-worker.js`; публично.
- Возвращает JavaScript UTF-8 с идентификатором текущей сборки, запретом промежуточного кэширования и областью `/`.
- При установке сохраняет только общедоступную оболочку; при активации удаляет кэши старых версий; обрабатывает только `GET` своего источника.
- Machine JSON, защищённые страницы и ответы с персональными данными исключены. Для навигации без сети используется `/offline`.

## Пограничные перенаправления страниц

- Нет сессии: Rails отвечает `303` на `/login`; недействительные cookie удаляются.
- Вошедший пользователь на `/login` или `/register`: `303` в кабинет, подтверждение почты либо настройку ключа.
- Неподтверждённый пользователь на защищённой странице: `303` на подтверждение.
- Ограниченная сессия: `303` на настройку ключа.
- JSON сохраняется только для явно зафиксированных machine/protocol границ.
