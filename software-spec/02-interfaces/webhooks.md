# Вебхуки

## Входящие

Clean Pay не содержит входящей webhook-операции. Telegram-аутентификация приходит как browser OIDC callback, а не Telegram Bot webhook.

## Исходящие

Clean Pay не отправляет webhook-уведомления. В тестовой инфраструктуре Remnashop может использовать `REMNAWAVE_WEBHOOK_SECRET`, а почтовый mock — передавать сообщения в локальный `smtp-log`; это поведение внешних тестовых контейнеров, не интерфейс приложения Clean Pay.

## Требование совместимости

Новая реализация на любом языке не должна вводить webhook endpoints или deliveries как будто они существовали в текущем продукте. Если они понадобятся, это отдельное архитектурное решение и версия контракта.
