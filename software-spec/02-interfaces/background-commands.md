# Фоновые команды

## BG-001 Исполнитель очистки

- Тип: отдельный долгоживущий процесс, не зависящий от языка реализации приложения.
- Обязательный вход: `DATABASE_URL`.
- Интервал: `DATA_RETENTION_INTERVAL_SECONDS`, по умолчанию 21600, диапазон 300–86400.
- Политика: пять сроков хранения; security-аудит хранится не меньше INFO-аудита.
- Цикл: очистка → структурированный итог → heartbeat `/tmp/clean-pay-retention-heartbeat` → ожидание остатка интервала.
- Ошибка: журналировать и повторить на следующем интервале; процесс остаётся жив.
- Остановка: завершить текущий ограниченный пакет и закрыть соединение с БД.
- Удаляет только категории, перечисленные в `06-data/retention.md`; пользователей и платёжные данные не удаляет.

## BG-002 Исполнитель сверки платежей

- Тип: отдельный долгоживущий процесс, общающийся с приложением только через внутреннюю HTTP-операцию.
- Работает только при `PAYMENT_RECONCILIATION_ENABLED=true`; иначе фиксирует отключение и завершается с кодом 0.
- Нужен отдельный секрет длиной не менее 32 символов.
- Операция: настроенный URL либо `http://app:4000/api/internal/payments/reconcile`.
- Интервал: по умолчанию 30 секунд, диапазон 5–3600.
- Запрос: `POST`, `Accept: application/json`, секретный заголовок, тайм-аут 45 секунд.
- Успех: проверить ограниченную схему ответа, журналировать счётчики/ID ручной проверки, обновить `/tmp/clean-pay-reconciliation-heartbeat`.
- Не-2xx, неверный JSON или сеть: журналировать и повторить на следующем интервале.

## BG-003 Команда миграции и запуска

Production-запуск проверяет конфигурацию, сериализует применение миграций advisory-lock, применяет ожидающие миграции, проверяет итоговую версию схемы и запускает HTTP-процесс. Ошибка до HTTP-запуска завершает процесс ненулевым кодом; правила отката заданы в `06-data/migrations.md`.

## BG-004 Исполнитель сквозных тестов

Тестовый исполнитель поднимает dependency stack, ждёт готовности, выполняет наборы тестов и поддерживает явный reset. Это инструмент контролируемой тестовой среды, а не production-поведение продукта.

## Модель планирования

Нет cron-выражений и долговечной очереди сообщений. Production-фон — отдельные долгоживущие сервисы с интервальными циклами и heartbeat healthcheck.
