# События

## Внешняя шина событий

Брокера/шины сообщений нет; приложение не публикует и не потребляет Kafka/RabbitMQ/NATS/SQS-подобные события.

## Долговечные события аудита

Фактическими внутренними доменными/операционными событиями служат строки `AuditLog`: `action`, `severity`, необязательные `userId`, `ipHash`, `metadata`, `createdAt`. Имена действий являются стабильным словарём наблюдаемости из `07-operations/observability.md`.

## Структурированные события журнала

Журнал пишет JSON по одной записи на строку в production и читаемый консольный вид вне production. Имя события, категория, источник и очищенные метаданные — наблюдаемый эксплуатационный выход. Подробные auth-события включаются отдельной настройкой и всё равно не раскрывают секреты.

## События браузера и PWA

Frontend реагирует на `beforeinstallprompt`, события service worker `install`, `activate`, `fetch`, состояние online/navigation браузера и результаты WebAuthn. Это UI/runtime-события, а не распределённая шина.
