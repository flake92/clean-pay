# Браузер, WebAuthn и PWA

## Браузер как внешний участник

Браузер хранит и передаёт сеансовые cookie, выполняет перенаправления, церемонии WebAuthn, виджет Turnstile, установку PWA, операции с буфером обмена, Cache API, `localStorage`/`sessionStorage` и переход на URL платёжного провайдера. Это часть наблюдаемого контракта, а не внутренняя деталь клиентской реализации.

## Реестр браузерных внешних интерфейсов

| ID | Интерфейс | Условие | Наблюдаемый результат или отказ |
|---|---|---|---|
| BR-001 | Навигация по HTTP(S)-адресу из `payment_url` | успешное создание покупки или продления вернуло непустой URL | браузер покидает Clean Pay и открывает провайдера; при `null` остаётся в кабинете |
| BR-002 | `navigator.clipboard.writeText(subscription.url)` | пользователь нажал копирование URL живой подписки | при успехе показывается подтверждение; при исключении — управляемое сообщение об ошибке |
| BR-003 | `navigator.credentials.create` | сервер выдал параметры регистрации ключа доступа | браузер возвращает WebAuthn credential либо отказ/исключение пользователя или платформы |
| BR-004 | `navigator.credentials.get` | сервер выдал параметры входа по ключу доступа | браузер возвращает assertion либо отказ/исключение |
| BR-005 | `beforeinstallprompt.prompt()` | браузер передал отложенное событие установки | приложение запрашивает установку и учитывает выбор пользователя |
| BR-006 | Service Worker, Cache API и сетевой `fetch` | PWA зарегистрирована | публичная оболочка обслуживается по правилам кэша; чувствительные ответы не кэшируются |
| BR-007 | `localStorage` | создана платёжная операция или обработан возврат | сохраняется корреляция операции; недоступность хранилища не должна создавать ложный успех |
| BR-008 | `sessionStorage` | создаётся платёжная попытка или обнаружен Telegram WebApp | сохраняются ключ идемпотентности и признак WebApp в пределах вкладки |
| BR-009 | статический ресурс логотипа того же origin | настроен безопасный корневой путь `NEXT_PUBLIC_BRAND_LOGO_URL` | браузер запрашивает ресурс у Clean Pay; абсолютные URL, `//`, обратная косая черта и NUL конфигурацией запрещены |
| BR-010 | системные обработчики `mailto:` и `https://t.me/...` | пользователь выбрал канал поддержки | управление передаётся почтовому клиенту или Telegram |
| BR-011 | Telegram WebApp `openLink` | встроенная среда предоставляет API и нужно открыть внешний адрес | вызывается `openLink(url, {try_instant_view:false})`; при отсутствии API используется обычная браузерная навигация |

## WebAuthn

- RP ID выводится из имени узла публичного источника.
- Ожидаемый источник должен точно совпасть с публичным источником.
- Для регистрации и входа требуется проверка пользователя.
- Серверные вызовы-проверки одноразовые и живут 5 минут.
- Параметры и ответ учётных данных браузера передаются в JSON-формате WebAuthn; бинарные значения кодируются base64url.
- Успешная проверка создаёт/подтверждает локальную сессию; повтор, неверный challenge, origin, RP ID или подпись отклоняются.

## Browser storage платежей

`sessionStorage` сохраняет устойчивость ключа идемпотентности в пределах вкладки и попытки. `localStorage` сохраняет ссылку на возврат платежа: локальный ID операции и/или ID платежа провайдера, включая чтение устаревших ключей. Ошибка транспорта, `202`, `408`, `429`, `5xx` и недопустимый формально успешный ответ внешней системы не дают права генерировать новый ключ для той же попытки.

## Хранилище Telegram WebApp

`sessionStorage` отмечает, что текущая вкладка была открыта как Telegram WebApp. Клиентский API извлекает `initData`; отсутствие или ошибка SDK приводит к управляемому состоянию ошибки, а не к непроверенной идентичности.

## PWA

Манифест публикует название, иконки, автономный режим отображения и метаданные темы. Service Worker кэширует только разрешённые публичные статические ресурсы и автономную оболочку. API, приватные страницы, аутентификация, платежи, зависимые от cookie и иные чувствительные ответы не должны попадать в Cache API. При сетевой ошибке разрешён только заранее подготовленный безопасный автономный ответ.

## Установка

Интерфейс различает поддерживаемое браузером событие установки, iOS, Android, Telegram или иной встроенный просмотрщик и конкретные браузеры. Встроенному просмотрщику может быть предложено открыть обычный браузер. Эти ветви и русские подсказки должны быть точно восстановлены во frontend-спецификации и макете.
