# Промпт модуля подписки

Реализуй владельца тарифов и доступа внутри Ruby-монолита. Прочитай весь `01-modules/subscription/`, HTTP-021—HTTP-023 и HTTP-026—HTTP-030, Remnashop/Remnawave и UI кабинета. Сначала закрепи ответы для пустого, active, expired и unknown состояния. Затем реализуй точное сопоставление владельца live URL, устройства, reissue и promo. Не подставляй сохранённый URL при ошибке Remnawave и не скрывай успешные панели из-за ошибки соседней.
