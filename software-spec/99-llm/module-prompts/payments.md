# Промпт модуля платежей

Реализуй владельца платежных команд внутри Ruby-монолита. Прочитай весь `01-modules/payments/`, HTTP-024, HTTP-025, HTTP-031, HTTP-032, HTTP-038 и payment integration/data/locking docs. Начни с state machine и уникального `(user, idempotency key)`. Докажи фиксацию dispatch до внешнего вызова, повтор одного отпечатка, конфликт другого, восстановление после потери ответа, owner fence, page sync и переход в ручную проверку. Ни один неоднозначный исход не должен создавать вторую оплату.
