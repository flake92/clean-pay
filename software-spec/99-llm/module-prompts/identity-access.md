# Промпт модуля доступа и идентичности

Реализуй владельца правил идентичности внутри Ruby-монолита. Прочитай весь `01-modules/identity-access/`, HTTP-001—HTTP-020, HTTP-039, HTTP-041—HTTP-043, соответствующие разделы домена, данных и интеграций. Сначала создай тесты точного транспорта и всех cookie. Затем реализуй e-mail, локальные сессии/refresh-family, Telegram OIDC/WebApp, WebAuthn и безопасный merge. Обязательны конкурентные проверки, частичный успех подтверждения e-mail и запрет неоднозначного автоматического владельца. Не переходи к подпискам до прохождения всех сценариев модуля.
