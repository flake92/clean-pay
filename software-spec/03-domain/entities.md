# Сущности

| Сущность | Идентичность | Назначение и существенные свойства |
|---|---|---|
| Пользователь (`WebUser`) | локальный строковый ID; уникальные Remnashop ID, e-mail и Telegram ID | единый владелец; хранит профиль, флаги подтверждения/ожидания и доказательства незавершённого объединения |
| Сеанс (`WebSession`) | локальный ID; уникальный отпечаток текущего refresh-токена | локальная аутентификация, уровень доверия, сроки жизни, отзыв и единоличное хранение внешних токенов |
| Предшественник refresh-токена (`WebRefreshToken`) | локальный ID; уникальный отпечаток старого токена | ограниченное окно повторной выдачи того же преемника и обнаружение повторного использования |
| Ключ доступа (`WebAuthnCredential`) | локальный ID; глобально уникальный credential ID | открытый ключ, счётчик подписи, транспорт и сведения аутентификатора |
| WebAuthn-проверка (`WebAuthnChallenge`) | локальный ID; уникальная случайная проверка | одноразовая регистрация или аутентификация с окончанием срока |
| Telegram-состояние (`TelegramAuthState`) | уникальные отпечатки state, nonce и verifier | одноразовая OIDC-церемония, безопасный адрес возврата и необязательный текущий пользователь |
| Код e-mail (`EmailVerificationCode`) | локальный ID | отпечаток кода, число попыток, лимит, сроки отправки/истечения/использования |
| Подтверждение объединения (`AccountMergeConfirmation`) | локальный ID; уникальный отпечаток токена | явное согласие, обе внешние стороны, состояние, аренда и результат |
| Платёжная операция (`PaymentOperation`) | локальный ID; уникальная пара пользователь+отпечаток ключа; уникальный внешний ключ | долговечная команда, запрос, владелец, отправка, ответ и независимая сверка |
| Платёжная запись (`PaymentRecord`) | локальный ID; уникальный внешний ID; необязательный уникальный ID операции | снимок результата платежа, предложения и внешней хронологии |
| Состояние синхронизации (`PaymentHistorySyncState`) | ID пользователя | один курсор/поколение/аренда на пользователя |
| Запись аудита (`AuditLog`) | локальный ID | неизменяемый эксплуатационный факт с уровнем важности и очищенными метаданными |
| Событие лимита (`RateLimitEvent`) | локальный ID | долговечное событие антизлоупотребления для заданных ключа и действия |
| Настройка (`AppSetting`) | строковый ключ | произвольное JSON-значение конфигурации времени выполнения |
| Статус интеграции (`IntegrationStatus`) | локальный ID; уникальное имя сервиса | последний операционный снимок состояния, не источник фактической доступности |

Имена в скобках — стабильные логические имена модели данных. Реализация вправе назвать классы иначе, но обязана сохранить семантику и ограничения.
