# Связи и правила удаления

| Родитель | Дочерняя сущность | Кратность | Поведение при удалении родителя |
|---|---|---:|---|
| Пользователь | Сеанс | 0..N | каскадное удаление |
| Сеанс | Предшественник refresh-токена | 0..N | каскадное удаление |
| Пользователь | Ключ доступа | 0..N | каскадное удаление |
| Пользователь | WebAuthn-проверка | 0..N | каскадное удаление |
| Пользователь | Код e-mail | 0..N | каскадное удаление |
| Пользователь | Telegram-состояние | 0..N | ссылка на пользователя обнуляется |
| Пользователь | Подтверждение объединения | 0..N | каскадное удаление |
| Пользователь | Платёжная операция | 0..N | удаление пользователя запрещено, пока операция существует |
| Пользователь | Платёжная запись | 0..N | каскадное удаление |
| Пользователь | Состояние синхронизации | 0..1 | каскадное удаление |
| Пользователь | Запись аудита | 0..N | ссылка на пользователя обнуляется |
| Платёжная операция | Платёжная запись | 0..1 | ссылка записи на операцию обнуляется |

Каждый внешний платёжный идентификатор относится не более чем к одной платёжной записи. Каждая платёжная запись относится не более чем к одной операции, и каждая операция — не более чем к одной записи. Настройки, события ограничения частоты и статусы интеграций независимы от пользователей.
