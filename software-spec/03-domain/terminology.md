# Соответствие терминов

| Логический термин | Стабильное имя данных | Значение |
|---|---|---|
| пользователь | `WebUser` | локальный владелец идентичностей и данных |
| локальный сеанс | `WebSession` | аутентифицированный контекст браузера |
| предшественник refresh-токена | `WebRefreshToken` | доказательство ротации и льготного повтора |
| ключ доступа | `WebAuthnCredential` | passkey/WebAuthn credential |
| одноразовая проверка | `WebAuthnChallenge` | challenge регистрации либо входа |
| состояние Telegram | `TelegramAuthState` | OIDC state/nonce/PKCE и возврат |
| платёжная команда | `PaymentOperation` | идемпотентное намерение и его выполнение |
| платёжная проекция | `PaymentRecord` | локальный снимок внешнего платежа |
| подтверждение объединения | `AccountMergeConfirmation` | явное разрешение переноса владельца |
| магазин / upstream | Remnashop | тарифы, подписка, аутентификация и платежи |
| движок подписок | Remnawave | пользователи/подписочные URL/устройства |
| почтовый транспорт | SMTP | протокол исходящей доставки писем |
| тестовый почтовый ящик | Mailpit | принимающий SMTP и HTTP-интерфейс просмотра в тестовой среде |
| ограждение владельца | owner fence | повторная проверка неизменности внешнего владельца перед эффектом |
| аренда | lease | ограниченное во времени право исполнителя изменить операцию |
