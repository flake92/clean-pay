# Объекты-значения

| Значение | Каноническая форма и проверка |
|---|---|
| Нормализованный e-mail | удалить краевые пробелы, привести к нижнему регистру; пустое значение запрещено |
| Безопасный адрес возврата | один корневой относительный путь `/…`; запрещены внешний origin, начало `//`, `\\` и NUL |
| Идемпотентный ключ | текст UUID; в хранилище только криптографический отпечаток |
| Отпечаток запроса | детерминированная канонизация вида операции и всех значимых полей; одинаковый смысл обязан давать одинаковый отпечаток |
| Подтверждение предложения | сумма — десятичная строка не более чем с 8 знаками после точки; валюта — 2–12 заглавных латинских букв/цифр; версия — строка не длиннее 2048 символов |
| Отпечаток внешнего владельца | ключевой криптографический отпечаток идентификатора аккаунта; сравнивается в постоянное время |
| Одноразовый секрет | наружу передаётся открытое случайное значение, в базе хранится только криптографический отпечаток |
| Денежная сумма | до подтверждения — точное десятичное значение; при сохранении результата — максимум 12 цифр всего и 2 после точки |
| Время | абсолютный момент UTC; отображение в локальной зоне выполняет интерфейс |
| Курсор истории | непрозрачная строка провайдера; не разбирается и не конструируется Clean Pay |

Перечисления состояний приведены в `states.md`; они являются закрытыми множествами, а неизвестное внешнее состояние явно отображается в `UNKNOWN`, если такой вариант предусмотрен.
