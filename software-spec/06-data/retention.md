# Хранение и очистка

| Категория | Значение по умолчанию | Правило |
|---|---:|---|
| Истёкшие/использованные auth-state и challenges | 7 суток | удалить только уже непригодные записи старше горизонта |
| История предшественников refresh | после истечения grace, в пределах auth-горизонта | удалить, если льготный повтор уже невозможен |
| Истёкшие/отозванные сеансы | 90 суток | активные сеансы не удалять |
| Аудит уровня `INFO` | 180 суток | удалить строки старше горизонта |
| Аудит безопасности `WARN`/`ERROR` | 365 суток | хранить дольше обычного аудита |
| События ограничения частоты | 30 суток | удалить старые события окна/доказательства |
| Период запуска очистки | 21600 секунд | допустимая конфигурация 300..86400 секунд |

Очистка выполняется пакетно и должна быть повторяемой: прерывание не повреждает оставшиеся строки. Она не удаляет пользователей, активные сеансы, платёжные операции, платёжные записи, состояние синхронизации или незавершённые объединения. Изменение сроков задаётся конфигурацией из `02-interfaces/configuration.md` и требует оценки правовых/эксплуатационных последствий.
