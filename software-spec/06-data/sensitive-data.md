# Чувствительные данные

| Данные | Хранение | Журналирование/выдача |
|---|---|---|
| Web refresh, OIDC state/nonce/verifier, merge-токен, e-mail-код, WebAuthn challenge | только криптографический отпечаток | открытое значение не журналировать и не возвращать после первоначальной выдачи |
| Преемник refresh-токена | шифротекст только на короткую льготу | возвращать лишь предъявителю допустимого предшественника |
| Access/refresh Remnashop | шифрование с аутентификацией ключом `WEB_REFRESH_SECRET` | никогда не отдавать браузеру и не писать в журнал |
| Локальный access-токен | подпись/HMAC ключом `WEB_JWT_SECRET`, cookie HttpOnly | содержимое не журналировать |
| Пароль, Turnstile token, Telegram токены | только в памяти на время запроса | обязательная рекурсивная редактировка |
| IP-адрес | ключевой отпечаток отдельным секретом аудита/лимита | открытый IP не сохранять в доменных таблицах |
| WebAuthn public key | открытые двоичные данные | допустимо хранить; приватный ключ никогда не покидает аутентификатор |
| Метаданные аудита и снимки ошибок | очищенный список полей | удалить password, token, cookie, authorization, code, secret и чувствительные псевдонимы |

Все cookies имеют `HttpOnly`, явно заданные `Secure`, `SameSite` и путь `/`. Временные Telegram cookies живут только срок церемонии и очищаются после потребления. Секреты конфигурации не должны совпадать между назначениями и не принимают шаблонные/слабые значения в production.
