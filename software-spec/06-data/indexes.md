# Индексы и пути доступа

## Уникальные пути

Индексами уникальности покрываются e-mail, Telegram ID и Remnashop ID пользователя; текущий и исторический отпечатки refresh; Telegram state/nonce/verifier; credential ID и WebAuthn challenge; токен объединения; внешний ID платежа; внешний ключ операции; пара пользователь+идемпотентный ключ; связь записи с операцией; имя интеграции.

## Очереди и сроки

Обязательны индексы для:

- поиска сеансов по пользователю, access/refresh-срокам, срокам внешних токенов, отзыву и уровню доверия;
- очистки одноразовых сущностей по истечению/использованию/отправке;
- захвата объединений по состоянию и аренде;
- захвата платёжных операций по состоянию/аренде и очереди сверки по следующей попытке/аренде;
- синхронизации истории по следующей попытке/аренде;
- истории пользователя по локальной дате и по `(userId, upstreamCreatedAt, paymentId)`;
- аудита по пользователю, действию и времени;
- лимитов по `(key, action, occurredAt)`.

## Запрет избыточности

Не создаются отдельные неуникальные индексы, полностью дублирующие начальную часть уникального индекса для равенства. В частности, уникальные индексы e-mail, Telegram ID и внешнего ID платежа одновременно обслуживают равенство. Удаление избыточного индекса должно иметь короткий тайм-аут блокировки и полностью откатываться при занятой таблице.
