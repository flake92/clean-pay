# Функциональные возможности

| Код | Возможность | Наблюдаемый результат |
|---|---|---|
| CAP-01 | Определение способа входа | система сообщает, существует ли адрес и доступен ли быстрый вход |
| CAP-02 | Регистрация и вход по почте | создаётся или возобновляется учётная запись и пользовательская сессия |
| CAP-03 | Подтверждение и изменение почты | адрес становится подтверждённым либо сохраняется ожидающий адрес |
| CAP-04 | Вход и связывание через Telegram | Telegram-идентичность связывается с единым пользователем |
| CAP-05 | Ключи доступа | регистрация, вход, просмотр и безопасное удаление ключей |
| CAP-06 | Управление способами входа | связывание и объединение учётных записей без потери владельца данных |
| CAP-07 | Каталог тарифов | публичные и персональные предложения с вариантами длительности и оплаты |
| CAP-08 | Управление подпиской | состояние, ссылка подключения, устройства, перевыпуск и промокод |
| CAP-09 | Покупка и продление | одна подтверждённая платёжная команда на одну пользовательскую попытку |
| CAP-10 | История и результат платежа | локально устойчивое отображение результата и фоновая сверка неопределённого исхода |
| CAP-11 | Поддержка | опубликованные контакты и ссылки помощи |
| CAP-12 | Устанавливаемое приложение | установка, обновление и безопасная автономная страница |
| CAP-13 | Эксплуатационный контроль | проверки жизни/готовности, аудит, очистка и восстановление |
