# Общесистемные правила

1. Внешние пути, методы, поля, коды состояния, заголовки, cookie, перенаправления и пользовательские подписи сохраняются без молчаливого изменения.
2. Изменяющий запрос из браузера принимается только от доверенного адреса приложения; запрос с телом должен быть JSON, кроме явно перечисленных операций без тела.
3. Пользовательская сессия имеет короткую проверяемую часть и долгую одноразово обновляемую часть. Обнаруженное повторное использование старой части завершает связанные сессии.
4. Ограниченная сессия не даёт общий доступ: пользователь должен завершить настройку ключа доступа.
5. Пользователь без подтверждённой почты и без Telegram не может выполнять защищённые операции, кроме подтверждения почты и выхода.
6. Одноразовые подтверждения, состояния входа и криптографические вызовы не могут быть успешно использованы повторно.
7. Один внешний идентификатор, адрес почты или Telegram-идентификатор не может одновременно принадлежать двум итоговым пользователям.
8. Клиент не управляет адресом возврата после оплаты: его формирует и проверяет система.
9. Повтор одной платёжной попытки не должен создать второй платёж; другой запрос с тем же ключом отклоняется.
10. Если платёж мог быть отправлен, но ответ неизвестен, система сохраняет неопределённый исход и выясняет результат, не создавая новую оплату автоматически.
11. Уже подтверждённый локальный результат платежа остаётся доступным при временной недоступности внешней системы.
12. Ссылка подключения берётся из фактической системы выдачи доступа; неоднозначная или устаревшая ссылка не показывается.
13. Секреты, пароли, токены, коды и полные сетевые идентификаторы не выводятся в журналы.
14. Устанавливаемое приложение не сохраняет в общий кэш персональные страницы или ответы API.
