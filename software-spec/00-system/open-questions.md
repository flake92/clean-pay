# Вопросы и контрольные неопределённости

## Текущий статус

Блокирующих открытых вопросов нет. Все ранее обнаруженные неопределённости либо разрешены наблюдаемым поведением, либо оформлены как принятые решения совместимости. Новая реализация не получает права менять эти решения молча.

## Подтверждённые решения

| ID | Подтверждённое поведение |
|---|---|
| OQ-001 | Legacy `/api/me`, `/api/logout` и `/api/bff` сняты; единственный браузерный вход — resourceful Rails routes |
| OQ-002 | Зафиксированы 42 канонические операции; HTTP-039/040 поглощены session resource без aliases |
| OQ-003 | Пользовательский `return_url` для покупки не принимается; сервер формирует и проверяет собственный pending URL |
| OQ-004 | Capability v1 и recovery строго проверяются; `404` capabilities включает legacy history fallback |
| OQ-005 | Неиспользуемые физические модели сохраняются как часть физической схемы, но не объявляются продуктовой функцией |
| OQ-006 | Redis rate-limit и readiness namespaces/TTL подтверждены; полный command matrix описан |
| OQ-007 | Обычные входы — Rails forms/strong parameters; JSON остаётся у WebAuthn и machine endpoints |
| OQ-008 | Приоритет query/storage ссылок возврата платежа и sessionStorage-idempotency подтверждён |
| OQ-009 | Публичная готовность без свежего успешного кэша — `degraded`; shared cache TTL 120 с порогом свежести 90 секунд |
| OQ-010 | Исторические документы не переопределяют текущее наблюдаемое поведение |
| OQ-011 | Mailpit — необязательная dev/test readiness-зависимость, не production SMTP-провайдер |
| OQ-012 | Текущая русская UI-копия является внешним контрактом до отдельного решения о локализации |
| OQ-013 | Текущее расхождение OIDC mock `username` против читаемого `preferred_username` сохраняется: username в mock-сцене `null`; исправление возможно только версией mock-контракта |
| OQ-014 | Проверенный Mailpit закреплён digest `sha256:37a38e9338cd7e89dfeb487f37b02ebfcd9cb23111bed2d345e79d37d6dd6`; prestage обязан отказаться от `latest` |

## Результаты контрольных вопросов

| ID | Проверенный результат |
|---|---|
| CQ-001 | Все 30 Remnashop операций сверены по формированию, runtime-разбору, закреплённой совместимой версии и тестам |
| CQ-002 | Все прямые, косвенные и test/spec-сервисы перечислены по отдельности, включая SMTP, Bot API, provider, worker/scheduler и хранилища |
| CQ-003 | Карточки входов проходят re-baseline под Rails forms; прежние JSON/BFF доказательства сброшены |
| CQ-004 | Сохранены 38 current и 38 mockup снимков, дизайн-токены, 19 PAGE-карточек и отчёт отклонений |
| CQ-005 | Clean-room аудит подтвердил отсутствие зависимости нормативных разделов от удаляемого исходного кода |
| CQ-006 | Описательный текст приведён к русскому; английский оставлен для точных wire identifiers, общепринятых протоколов и имён продукта |
| CQ-007 | В итоговых матрицах нет незакрытых статусов; ограничения физических проверок явно отделены от неопределённости контракта |
| CQ-008 | OQ-013 закреплён ADR-001 |
| CQ-009 | OQ-014 закреплён ADR-002 |

Если при реализации появится новый неустановимый факт, он должен быть оформлен новым вопросом и решением; текущий анализ не содержит таких блокирующих фактов.
