# ADR-002. Закрепить проверенный образ Mailpit

## Решение

В новом prestage использовать `axllent/mailpit@sha256:37a38e48e9338cd7e89dfeb487f37b02ebfcd9cb23111bed2d345e79d37d6dd6`, а не плавающий тег `latest`.

## Причина

SMTP, HTTP API, webhook и readiness Mailpit являются частью проверяемой тестовой обвязки. Плавающий образ не позволяет воспроизвести тот же контракт после удаления старого приложения.
