# Отклонённые варианты

- Не считать `success|fail|pending` в URL возврата источником результата платежа: источник истины — проверенный серверный статус.
- Не использовать сохранённый Remnashop URL подписки как запасной при неподтверждённом сопоставлении Remnawave.
- Не объединять внешних владельцев по неоднозначному совпадению.
- Не повторять платёж после неопределённого исхода как новую покупку.
- Не включать Remnashop, SMTP, Telegram Bot API или платёжного провайдера внутрь будущего монолита: они остаются внешними границами.
- Не удалять исходники или контейнеры автоматически по факту появления спецификации.
