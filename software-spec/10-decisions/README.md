# Архитектурные решения

Раздел фиксирует только решения, необходимые для сохранения совместимости. Он не выбирает Rails, ORM, структуру классов или внутреннее устройство будущего Ruby-монолита.

- `accepted/` — принятые по наблюдаемому поведению решения, обязательные до версионированного изменения контракта.
- `pending/` — вопросы, которые нельзя установить из источников и которые блокируют совместимость.
- `rejected/` — варианты, явно отвергнутые анализом.

На дату среза блокирующих решений в `pending/` нет.
