# Транзакции, блокировки и конкурентный доступ

| Сценарий | Блокировка/ограждение | Обязательный результат |
|---|---|---|
| Ротация refresh | строка сеанса; уникальные отпечатки; повтор чтения после конкурентной фиксации | один новый токен; конкурентный запрос в grace получает того же преемника |
| Удаление passkey | все credentials пользователя | нельзя одновременно удалить два последних ключа |
| Объединение | оба пользователя в стабильном порядке, затем платёжные границы | перенос целиком либо полный откат; итоговый владелец перепроверен |
| Передача внешних токенов | пользователь, затем активные сеансы по ID | ровно один хранитель комплекта после фиксации |
| Платёжная команда | уникальность пользователь+ключ; claim hash+lease; owner fence | один dispatch; устаревший исполнитель не завершает чужой claim |
| Сверка | отдельные claim/lease от foreground-dispatch | поздний foreground и worker не перезаписывают друг друга |
| История | пользователь+generation+cursor+claim/lease | одна страница фиксируется один раз; старое поколение отвергается |
| Подтверждение объединения | token hash, status и lease | один перенос; истёкшую аренду можно безопасно перезахватить |

Максимальное ожидание блокировки и транзакции для ротации сеанса — 5 и 15 секунд; для передачи токенов — 5 и 20 секунд. Превышение возвращает контролируемую временную ошибку и не оставляет частичную запись.

Сетевые обращения выполняются вне длительных row-lock транзакций. Право завершить результат доказывается непрозрачным токеном захвата, хранимым как отпечаток; одного неистёкшего времени аренды недостаточно. Все множественные блокировки берутся в детерминированном порядке ID для снижения риска deadlock.
