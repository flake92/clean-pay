# Фоновые задания

## Сверка платежей и продолжение истории

Задание включается отдельным флагом. Внешний цикл с периодом 5..3600 секунд (по умолчанию 30) отправляет `POST` на закрытую внутреннюю HTTP-операцию, предъявляя отдельный секрет. Тайм-аут клиента цикла — 45 секунд. Один вызов обрабатывает 1..100 элементов (по умолчанию 10); бюджет каждой сверки и шага истории — 12 секунд.

В одном пакете исполнитель:

1. захватывает доступные `OUTCOME_UNKNOWN` операции независимой арендой;
2. наблюдает состояние у провайдера и завершает, откладывает либо переводит операцию в ручное вмешательство;
3. захватывает не более одной следующей страницы истории на состояние синхронизации;
4. фиксирует результат только при совпадении токена аренды, поколения и внешнего владельца.

Ошибка отдельного элемента не отменяет остальные. Ошибка всего вызова журналируется, следующий период повторяет работу; вслепую повторно отправлять покупку запрещено. После успешного пакета обновляется heartbeat-файл `/tmp/clean-pay-reconciliation-heartbeat`. Он считается свежим не дольше `max(120, 2 × interval + 60)` секунд.

## Очистка хранения

Отдельный обязательный цикл запускается каждые 300..86400 секунд, по умолчанию 21600. Он применяет только правила из `06-data/retention.md`, считает удалённые строки по категориям, журналирует итог и обновляет `/tmp/clean-pay-retention-heartbeat`. Допустимый возраст heartbeat — интервал плюс 600 секунд. Сбой не останавливает приложение и повторяется в следующий цикл; частично выполненная очистка безопасна для повторения.

## Запуск и остановка

Каждый цикл выполняет первый запуск без одновременного наложения второго экземпляра в том же процессе, корректно реагирует на `SIGTERM`/`SIGINT` и не начинает новый пакет после сигнала остановки. Долговечная аренда защищает от параллельных контейнеров. Heartbeat создаётся атомарной заменой файла, а не дописыванием.
