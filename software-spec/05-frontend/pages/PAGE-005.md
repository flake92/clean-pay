# PAGE-005 — Подтверждение e-mail существующей сессии

## Маршрут и доступ

`GET /verify-email`, AppShell. Требует сессию; используется при неподтверждённом e-mail или после его смены. Profile загружается HTTP-004.

## Структура

Page header `Подтверждение e-mail`; карточка текущего адреса и его статуса; форма кода `000000` с confirm; отдельная форма запроса/повтора письма с e-mail; Message/Tag состояния; доступные переходы в профиль и кабинет согласно результату.

## Операции

Запрос письма → HTTP-007; подтверждение кода → HTTP-008; профиль/актуальный статус → HTTP-004. Если адрес уже подтверждён, форма кода заменяется success-состоянием. Если подтверждение принято, но синхронизация аккаунта ещё не завершена, показывается `account-sync-pending`, выполняется контролируемое обновление, успех не подменяется ошибкой.

## Состояния

Profile loading/error; unverified idle; request pending/success/error; confirm pending/invalid/expired/success; already verified; account sync pending. Две формы не должны одновременно отправляться. Серверный публичный error имеет приоритет над русским fallback.

## Адаптивность и приёмка

В desktop карточка располагается в AppShell с sidebar; mobile — одна колонка под topbar. Эталоны PAGE-005 фиксируют verified-сцену; unverified-сцена должна быть покрыта fixture. Labels, alerts и focus после submit обязательны.
