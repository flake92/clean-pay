# PAGE-001 — Главная

## Маршрут и доступ

`GET /`. Доступна гостю и пользователю. Оболочка AppShell: фиксированная верхняя панель, desktop-sidebar, основная область и footer. Сессия меняет доступность действий, но не композицию hero.

## Структура сверху вниз

1. Page header-card: eyebrow `Clean Pay`, H1 `Web-кабинет для оплаты и управления подпиской`, пояснение о платежах, продлении и VPN-доступе.
2. Справа в hero: primary `Войти` и outlined `Тарифы`.
3. Сетка четырёх одинаковых action-card: `Кабинет / Подписка`, `Профиль / Аккаунт`, `Привязать Telegram / Связь аккаунта`, `Поддержка / Помощь`; у каждой label, крупный title, описание и контурная иконка.
4. Footer с малым логотипом и `Clean Pay`.

## Действия и переходы

`Войти` → `/login`; `Тарифы` → `/tariffs`; карточки → `/cabinet`, `/profile`, `/link-account`, `/support`. Верхняя и боковая навигация подчиняется `navigation.md`. Protected destination вправе перенаправить гостя на `/login` с безопасным return path.

## Состояния и адаптивность

Контент статичен; API-ошибок внутри hero нет. Desktop: sidebar 300 px, hero в одну строку, четыре карточки в ряд. Mobile: sidebar скрыта, topbar содержит hamburger/logo/ellipsis, hero actions остаются в строке, пока помещаются, и переносятся/становятся вертикальными на узком mobile `≤360 px`; карточки идут по одной. Контент не должен вызывать горизонтальную прокрутку при 320 px.
