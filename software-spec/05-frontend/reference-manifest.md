# Манифест визуальных эталонов

## Назначение и границы

Каталог `reference/current` хранит наблюдаемый внешний вид действующего приложения. Это не иллюстрации и не предложение нового дизайна. Снимки сделаны из реально запущенного приложения 22 июля 2026 года при масштабе браузера 100 %. Формат файлов — JPEG; расширение соответствует фактическому содержимому. Контроль целостности находится в `reference/current/SHA256SUMS`.

Desktop снимался при заданном окне `1440×1000`, mobile — `390×844`. На страницах с собственной полосой прокрутки фактическая полезная область снимка равна `1425×990` и `375×812`: это наблюдаемое вычитание полосы прокрутки, а не иной breakpoint. Снимок фиксирует первый экран; длинные страницы дополнительно задаются структурой и порядком секций в карточке PAGE.

## Проверенное сквозное состояние

Для получения закрытых экранов фактически пройдена цепочка: регистрация локального пользователя → письмо через SMTP → получение письма в Mailpit → ввод шестизначного кода → bootstrap-сессия → экран Passkey → отказ от необязательного Passkey → полный кабинет → явный выход. Снимки закрытых страниц получены внутри этой сессии; публичные формы после выхода. Ни один внешний платёж и ни одна реальная Telegram-операция не выполнялись.

## Критерий использования при переносе

Новая реализация проходит визуальный барьер только если при той же сцене и viewport совпадают: shell, порядок секций, размеры ключевых блоков, переносы заголовков, видимость управляющих элементов, цвета, типографика и responsive-переход. Само наличие 38 файлов не закрывает visual gate: требуются карточки всех PAGE, воспроизводимые fixtures и отчёт сравнения нового UI с этими эталонами.
